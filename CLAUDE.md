## Project overview

This is a Jekyll-based academic personal website for Yue Huang (etherial-h), hosted on GitHub Pages. It was forked from the [academicpages](https://github.com/academicpages/academicpages.github.io) template, which itself derives from the [Minimal Mistakes](https://mmistakes.github.io/minimal-mistakes/) Jekyll theme.

## Local development

```bash
# Install Ruby dependencies (delete Gemfile.lock first if you get errors)
bundle install

# Start local dev server with live reload (uses _config.dev.yml overrides)
bundle exec jekyll liveserve

# Build for production
bundle exec jekyll build

# Uglify JavaScript (rarely needed)
npm run build:js
```

The dev server runs at `http://localhost:4000`. The `_config.dev.yml` overrides the production `_config.yml` to set `url: http://localhost:4000`, disable analytics, and output expanded (uncompressed) CSS.

There are no tests in this project — it's a static site generated by Jekyll.

## Publication schema

Publications are stored as markdown files in `_publications/`, one file per paper. File naming convention: `YYYY-MM-DD-short-slug.md`. They are sorted by date (newest first via `reversed` in the listing loop).

### Frontmatter fields

| Field | Required | Description |
|-------|----------|-------------|
| `title` | Yes | Paper title (double-quoted string) |
| `collection` | Yes | Always `publications` |
| `permalink` | Yes | URL path: `/publication/YYYY-MM-DD-slug/` |
| `date` | Yes | `YYYY-MM-DD` — determines sort order |
| `venue` | Yes | Journal or conference name (single-quoted to avoid YAML escaping) |
| `authors` | Yes | Full names, comma-separated. **Yue Huang** wrapped in markdown bold (`**Yue Huang**`) |
| `link` | Yes | External URL to the paper (IEEE Xplore, arXiv, conference PDF, etc.) — clicking the title navigates here |
| `citation` | No | Full citation string (HTML-escaped). Retained in frontmatter for reference but NOT displayed on the page |
| `paperurl` | No | Legacy field — not displayed for publications (citation and excerpt blocks are hidden for the publications collection) |
| `excerpt` | No | Do NOT include — excerpts are hidden for publications |

### Body content

Publication files have **empty body** — everything is in frontmatter. The description and recommended citation text that existed in the template placeholders were removed.

### Rendering (via `_includes/archive-single.html`)

Each publication renders on `/publications/` as:

1. **Serial number** — `{{ forloop.index }}.` before the title
2. **Title** — linked to `post.link` (the external paper page, e.g. IEEE Xplore). A small permalink icon (🔗) next to it links to the internal `post.url`
3. **Authors** — rendered via `markdownify` so `**Yue Huang**` produces bold
4. **Venue** — "Published in *Venue*, Year"
5. **No excerpt, no citation, no body text** — these are gated behind `{% if post.collection != 'publications' %}` in the template

### Adding a new publication

Create a file `_publications/YYYY-MM-DD-short-slug.md`:

```markdown
---
title: "Paper Title Here"
collection: publications
permalink: /publication/YYYY-MM-DD-short-slug
date: YYYY-MM-DD
venue: 'Journal or Conference Name'
authors: 'First Author, **Yue Huang**, Other Author'
citation: 'Full citation with &amp;quot; etc.'
link: 'https://doi.org/...'
---
```

- Sort order = date descending (newest first). Use appropriate month/day so papers within the same year sort correctly.
- The `citation` field is for reference — it won't appear on the page but is good practice to keep.
- Navigation link in `_data/navigation.yml` must be uncommented for the Publications tab to appear.

## Architecture

### Content layer (editable by the site owner)

- **`_config.yml`** — Site-wide settings: author info, social links, collections config, pagination, plugins, etc. This is the primary configuration file.
- **`_pages/`** — Top-level pages (About, CV, Publications, Talks, Teaching, etc.). Each is a `.md` file with YAML frontmatter.
- **`_posts/`** — Blog posts in `YYYY-MM-DD-title.md` format.
- **`_publications/`** — Individual publication entries as markdown files. Can be auto-generated from `markdown_generator/publications.tsv`.
- **`_talks/`** — Individual talk entries. Can be auto-generated from `markdown_generator/talks.tsv`.
- **`_portfolio/`** — Portfolio project entries.
- **`_data/`** — YAML data: `navigation.yml` (nav menu items), `authors.yml` (co-author bios for publications), `ui-text.yml` (i18n UI labels in multiple languages).
- **`_drafts/`** — Unpublished draft posts.
- **`images/`** — Site images (profile photo, favicon tiles, etc.).
- **`files/`** — Downloadable PDFs and other files served at `/files/...`.
- **`talkmap/`** — Interactive Leaflet map of talk locations. Generated by `talkmap.py`.

### Theme layer (rarely modified)

- **`_layouts/`** — Liquid HTML templates. Hierarchy: `compress.html` → `default.html` → `single.html` (or `talk.html`, `archive.html`, `splash.html`). `default.html` assembles `<head>`, masthead, content, and footer. `compress.html` minifies the final HTML output.
- **`_includes/`** — Reusable Liquid partials: `head.html`, `masthead.html`, `footer.html`, `sidebar.html`, `author-profile.html`, `scripts.html`, `analytics.html`, `comments.html`, `social-share.html`, etc. The `head/custom.html` and `footer/custom.html` are extension points for adding custom HTML.
- **`_sass/`** — SCSS partials following Minimal Mistakes architecture. Entry point is `assets/css/main.scss`. Includes vendor libraries: Breakpoint (media queries), Susy (grid system), Font Awesome, and Magnific Popup. Theme partials cover: variables, mixins, reset, base, utilities, animations, tables, buttons, notices, masthead, navigation, footer, syntax highlighting, forms, page layouts, archive layouts, sidebar, and print styles.
- **`assets/`** — Compiled/static frontend assets: `css/main.scss` (compiled by Jekyll), `js/` (jQuery-based plugins), `fonts/`.

### Content generation scripts

- **`markdown_generator/`** — Python scripts and Jupyter notebooks to batch-generate publication and talk markdown files from TSV data. `publications.tsv` → individual `_publications/*.md`, `talks.tsv` → individual `_talks/*.md`.
- **`talkmap.py`** / **`talkmap.ipynb`** — Generate `talkmap/org-locations.js` from talk frontmatter for the Leaflet map.

### How content renders

1. Jekyll reads content files (pages, posts, collections) and parses their YAML frontmatter + markdown body.
2. The frontmatter's `layout` field selects a template from `_layouts/`.
3. `_config.yml` `defaults` section assigns default layouts and settings per collection type.
4. Layouts use Liquid tags to pull in `_includes/`, site data from `_data/`, and `site.*` / `page.*` variables.
5. SCSS in `assets/css/main.scss` is compiled to CSS during build. JavaScript is pre-minified (via npm `uglify` script) and served as-is.
6. Plugins (`jekyll-paginate`, `jekyll-sitemap`, `jekyll-gist`, `jekyll-feed`, `jekyll-redirect-from`) run at build time.
